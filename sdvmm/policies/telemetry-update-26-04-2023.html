    Update Overview
    As of Stardew Valley Mod Manager build v230401_indev_260423_149P and newer, the Stardew Valley Mod Manager now collects additional data as a part of the opt-in telemetry program. We have made the executive decision not to re-prompt users if they'd like to stay a part of the telemetry program with this change as the new  information we're collecting is still anonymous and cannot be tied to an individual, and our original policy reflects that RWE Labs may add new statistics and information to telemetry in the future without notice.

    New Information Collected
    We're now collecting the following new information as a part of this update:
    - The version number of the Stardew Valley Mod Manager being used 

    Additional Policy Update
    We've also tried our best to clarify our stance on collecting new information after the user has selected to opt-in and how we justify whether we should ask the user if they'd like to continue in the program after new information starts being collected. We see it as follows:

    - If the data remains within the same scope of the original policy (e.g. the data is anonymous and cannot be tied to an individual) then we will assume that the user wishes to continue to stay opted-in to the program. The way we see it, adding the version number of their mod manager does not violate the user's privacy and is not traceable back to them, therefore, it fits the scope of the original policy and does not require a confirmational re-opt-in.

    - If the data is different to the scope of the original policy (e.g. if we were to start collecting hardware IDs, usernames, etc. that could be tied to an individual or reveal information about the individual) then we must announce these changes through the telemetry onboarding window in-app, and give the user an option to change their opt-in status the first time they use the app after updating to a version with the changes.
